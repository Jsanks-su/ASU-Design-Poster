# Prompt Templates

Use these as structures, not fill-in forms. Replace bracketed fields with concrete decisions from the brief and the style route.

Default to a vertical poster with visible poster typography unless the user explicitly asks for a horizontal crop or text-free post-production image. For ChatGPT Image 2 tests, include a main title treatment plus context-appropriate supporting copy. Small release-like microtype, bilingual fragments, invented emblems, logo-like marks, icons, pictograms, seals, and footer metadata are allowed when they improve hierarchy and atmosphere. Avoid automatic full cast-credit blocks, false real release claims, and real brand/studio logos unless the user asks for release simulation or supplies those assets.

Before sending any prompt to an image API, replace every bracketed placeholder and remove internal decision language. The prompt must be generation-ready. Do not use these templates to run automatic regenerate-after-review loops; fix prompt problems before generation.

## Variant Pattern

When offering alternatives, vary the design hypothesis only if the content supports it. Do not vary the visual family mechanically if the content asks for a narrower world:

1. Faithful/congruent: expected genre or subject language, sharpened by craft.
2. Counterpoint: a deliberate mismatch that reveals a new tension.
3. Argument-led: one symbolic thesis, often graphic, material, or typography-led.
4. Popular-readable: clear subject, bold contrast, broad appeal without template habits.
